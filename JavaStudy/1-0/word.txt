■ IT
IT
	InformationTechnology（情報技術）

	例：コンピューター、ソフトウェア
ICT
	Information and Communication Technology（情報通信科学）
	情報技術で人々が繋がる技術
IoT
	Internet of Things

■ OS
OS
	OperatingSystem
	アプリケーションとハードウェアの仲介役
	ユーザーにインターフェースを提供し、ハードウェアに処理の指示を与える
Windows
	マイクロソフトが開発したOS
macOS (Mac OS X)
	Appleが開発したOS
	Unixの特徴を受け継いでいる
UNIX
	様々なOSの先祖
	企業や研究所で利用される
	ライセンス契約で使用された
Linux
	オープンソースOS
	Unixを基に開発された
	誰もが使えるOSとして開発され、Unixよりも縛りが少ない
	
■ 言語・環境
マークアップ／フロントエンド
	マークアップ：タグで文章の構造を表現する方式
		例：HTML、XML
	フロントエンド：ユーザーに見える部分
		例：Webページ、WebサイトのUI
		HTML/CSS, JavaScriptで作られる
		対義語：バックエンド：ユーザーからは見えない部分
HTML
	Hyper Text Markup Language
	Webページを構築するルールの1つ
	Webページの内容（文章、画像）を表示
CSS
	Cascading Style Sheets
	Webページの見た目を定義するルール
JavaScript
	Webページの動作を定義する言語
	例：アニメーション、UIの挙動、
jQuery
	JavaScriptを簡易的に書けるライブラリ
サーバーサイド
	クライアントにWebページやサービスを提供する側
.NET（C#, VB）
	Windowsアプリケーション（ASP.NET）を作れる言語
	C#：C言語の派生、Webアプリ開発でも
	VB：VBA（Excelの言語）の派生、自然言語
Java
	どんな環境でも実行できる言語
	オブジェクト指向言語の代表格
	JavaVM（Java仮想マシン）で、どの環境でもJavaを実行できる
Java SE
	Javaの基本機能をまとめたもの
	小規模（パソコン上のソフト）
Java EE（Jakarta EE）
	JavaSEにサーバーサイド機能を追加したもの
	大規模（Webサイト、Webアプリ）
Perl
	スクリプト言語
	コンパイル不要
	CGI（CommonGatewayInterface）（外部サーバー内でプログラムを実行する仕組み）の構築に用いられる
	Webアプリ、文書系のプログラム開発に用いられた
	Unix,Linuxで使用される
PHP
	Hypertext Preprocessor
	Web制作、Web開発用のスクリプト言語
Ruby
	製作者：まつもとゆきひろ
	バックエンド（Webサーバ側）の開発で利用されるスクリプト言語

■ SQL
SQL
	Structured Query Language
	データベースを操作するための言語
DB2
	IBMのデータベース
MySQL
	Webシステム（インターネットを利用したシステム）で利用されるSQL
Maria DB
	MySQLから派生したデータベース
Oracle DB
	Oracleが開発したデータベース
	世界初の商用DB
PostgreSQL
	MySQLに似たもの
SQLite
	軽量なデータベースに向いているSQL
SQL Server
	Windowsのデータベース
CRUD
	システムの基本的な機能
	Create(追加), Read(参照), Update(変更), Delete(削除)
トランザクション
	処理単位
	データベースで使われる
	例：業務開始までの処理単位「出社→タイムカードスキャン→業務開始」
コミット
	トランザクションの処理を確定させる処理
ロールバック
	トランザクションを中断し、元の状態に戻す
	やりかけの処理を戻す
排他制御（排他ロック）
	同時アクセスを防ぐ仕組み
	アクセス中は他がアクセスするのを禁止
楽観ロック
	更新がめったに起きない前提の排他制御
	データが初回取得時と中身が同じか、同じバージョンかで更新の可否を決める
悲観ロック
	更新が頻繁に起こることが前提の排他制御
	トランザクション（取得から更新まで）が終了するまで、データをロック
	更新完了後にロック解除が必要なので、管理者がロック解除をするなど、開発コストが高い
実行計画
	SQLで実行されるプログラムの手順
	SQLで使われる値をインデックスやテーブルでまとめたもの
チューニング
	良い結果になるように調整
	例：中身を書き換える、別のものに乗り換える
パフォーマンス
	性能
スロークエリ
	速度が遅いSQL
	無駄なデータ操作が行われてるので、解消すべき事案
クエリログ
	SQLの実行履歴
IDE（統合開発環境）
	integrated development environment
	開発に必要な機能を揃えたツール
	搭載機能：コンパイラ、デバッグ、
Eclipse
	Java向けのIDE
	NetBenasと似てる
NetBeans
	オラクルのIDE
	Java向けのIDE
SpringToolSuite
	Spring（Web開発向けのJavaライブラリ）用のIDE
	EclipseやVSCodeに導入することも可能
DBツール
	データベースを操作するツール
	例：SQLをGUIベースで操作
A5:SQL Mk-2
	SQLをGUIベースで操作できるツール
pgAdmin
	PostgreSQLをGUIで管理できるツール
DBeaver
	様々なSQLをGUIを操作できるツール
MySQL Workbench
	MySQLをGUIで管理できるツール
Sequel Pro
	MySQLをGUIで管理できるツール
TeamSQL
	様々なSQLをGUIを操作できるツール

■ ファイル／ディレクトリ
ファイル
	コンピューター上の書類

拡張子
	ファイルの種類

ディレクトリ

■ ネットワーク
LAN
	Local Area Network
	狭い範囲での繋がり
WAN
	Wide Area Network
	広い範囲での繋がり
クライアント
	Webサービスを利用する人、コンピューター
	サーバーと連携し、データやWebページを取得/表示

	Webサービスを提供する人、コンピューター

■ 各種プロトコル
HTTP
	HyperText Transfer Protocol
	Web上の通信規格
	HTTPに従って通信が行われる
FTP
	File Transfer Protocol
	ファイル送受信の規格
HTTPS
	HTTPを暗号化したもの
SSL
	Secure Sockets Layer
	データを暗号化し、暗号鍵で元の状態に復元する仕組み
	HTTPよりも安全
SSH
	Secure Shell
	telnetを暗号化したもの
		telnet：サーバーやルータを遠隔操作する技術、規格
ポート、ポート番号
	ポート：コンピューターとネットワークの接続口
	ポート番号：サービス別で割り振られたポートの識別番号
HTTP、FTP、SSHの各ポート番号
	HTTP 80
	FTP 20, 21
	SSH 22
ファイアウォール
	通信の安全性を確かめる機能
	外部ネットワークからの危険な通信から守る
TCP
	Transmission Control Protocol
	安全で低速度な伝達コントロール規格
	HTTPの土台がTCP
	HTTPとは立ち位置が違う
		HTTP：アプリケーション層
		TCP：トランスポート層

■ IPアドレス
IPアドレス
	Internet Protocol address
	インターネットにおけるコンピューターの住所
	例：192.193.4.2
	 MACアドレス：ネットワーク機器の住所、データの受け渡し先
グローバルIPアドレス
	全世界で利用されるIPアドレス
プライベートIPアドレス
	限定された範囲でしか利用できないIPアドレス	

■ ドメイン
ドメイン
	ドメイン名の省略形、領域、範囲
	ドメイン名：IPアドレスを人が読めるようにしたもの
	例：example.com
DNS
	ドメイン名をIPアドレスに変換する仕組み
VPN	
	Virtual Private Network
	限られた人だけが通れる仮想の私用ネットワーク
ブリッジ
	ネットワークの接続先を振り分ける機器
	MACアドレスでハンネbつ
プロキシサーバー
	Webブラウザの代替
	クライアントの代わりにサーバーとやりとり
	匿名性が高まる、キャッシュでアクセスが早くなる

■ 電子メール
SMTP
	Simple Mail Transfer Protocol
	メール送信の規格
	SMTPサーバー：目的地までメールが経由する場所
POP
	Post Office Protocol
	受信されたメールのコピーを自分のパソコンで見る方式
IMAP
	Internet Message Access Protocol
	受診されたメールを直接閲覧する方式

■ セッションとクッキー
セッション
	開始〜終了までの処理単位
クッキー
	WebサーバーがWebブラウザに渡す情報
	アクセス履歴やサイト設定などが一定期間保存される
キャッシュ
	二度手間を防ぐ仕組み
	Webページのコピーを確保し、次回以降で使用され、初回よりも表示の手間が省ける

■ DB（データベース）について
インデックス
	データ検索を効率的に行うもの
	各データを要素（id、名前、日付など）で紐付け、データ操作を容易にする
	id=??で特定のデータを検索、名前でデータを並び替え

■ バージョン管理
Git
	分散型
	各コンピューターで管理
	複数人が共通で管理できるGitHubもある
	同期、細かい差分の保持で
SVN
	Apache Subversion
	集中型
	リモートのみ管理され、ユーザーはダウンロードして私用
	何度もリモートへのアクセスが必要、同期が苦手でコンフリクトがマージで問題が起こりやすい
デグレ
	デグレード（以前よりも悪い状態であること）の省略形
マージツール
	マージ：複数のものを合体する
	ファイルの変更を統合するツール
	例：txtファイルの文章を片方のファイルにコピー
	Winmerge：オープンソースのマージツール
